## B44 Game Architecture and Product Rules

- Every B44 game has an engine-free `*.Core` project using `Microsoft.NET.Sdk`. It sets `<B44EngineFreeCore>true</B44EngineFreeCore>`, carries no Godot package or assembly reference, and builds and tests without the engine. Godot projects are presentation/adaptation layers over that Core, not alternate owners of game rules or state.
- Scene controllers stay thin: translate engine input and output at the boundary, while rules, algorithms, state machines, and persistent game state live in Core behind concrete collaborators.
- This is a release game, not a prototype. Architecture and test coverage decisions must reflect that.
- While a game is unreleased, unreadable saves may reset and shape-breaking format changes do not require migrations. At 1.0 this flips: released saves are a compatibility surface and require versioned envelopes and migrations.
- Domain operations return per-game `*ActionResult` records that describe outcomes for the presentation layer. These shapes remain game-owned rather than entering B44.Common.
- Core receives wall-clock time through an injected BCL `TimeProvider` and randomness through `B44.Common.Interfaces.IRandomSource`; never call ambient `DateTime`, construct `System.Random`, or call Godot randomness from Core.
- Log through `B44.Common.Diagnostics.StructuredGameLogger` with game-owned categories and one Godot sink at the boundary. Declare scene node paths once in `*Paths.cs`, use engine-free value types in Core with one conversion bridge per type, and run AI/solver simulations through copied real game state rather than parallel rule implementations.
- Every B44 game is sunset-safe: no developer-operated server, account, advertisement, purchase, or online service may be required to start, progress, unlock content, save, reload, or finish. Required content ships in the client; optional-service failure cannot block play; online rewards receive deterministic local replacements; progress remains locally durable and exportable.
- Ads and purchases may accelerate but never gate core progression. Reward failure cannot cost the player, and paid currency must never buy repeated random rolls.
- Engine-free shared primitives move to B44.Common on their second cross-game occurrence. Do not copy package types back into a game or use the package as a general utility dump.
